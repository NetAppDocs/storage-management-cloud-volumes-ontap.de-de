Sie können Cloud Volumes ONTAP in Amazon Web Services (AWS) und Microsoft Azure mit  `Quick create` für eine schnellere Bereitstellung sowohl in Einzelknoten- als auch in Hochverfügbarkeitskonfigurationen (HA) bereitstellen. Dieser vereinfachte Prozess reduziert die Bereitstellungsschritte im Vergleich zur fortschrittlichen Methode, indem Standardwerte automatisch auf einer einzigen Seite festgelegt und die Navigation minimiert wird.

.Systemdetails
. *Cloud-Anbieterkonto/Anmeldeinformationen*: Die Konto- und Abonnementdetails werden automatisch anhand Ihres Console-Agenten ausgefüllt. Wenn Sie mehrere Konten oder Abonnements haben, wählen Sie das gewünschte aus. Falls kein Console-Agent verfügbar ist, werden Sie dazu aufgefordert,  https://docs.netapp.com/us-en/console-setup-admin/concept-agent-install-options.html["Erstellen Sie einen Konsolenagenten"^].
. *Name*: Geben Sie einen Namen für Ihr Cloud Volumes ONTAP-System ein (auch Clustername genannt). Die Console verwendet den Systemnamen (Clusternamen) zur Benennung des Cloud Volumes ONTAP-Systems und der VM-Instanz. Der Name wird auch als Präfix für die vordefinierte Sicherheitsgruppe verwendet, wenn Sie diese Option auswählen.
. *ONTAP-Anmeldeinformationen*: Dies sind die Anmeldeinformationen für das Cloud Volumes ONTAP-Administratorkonto. Sie können diese Anmeldeinformationen verwenden, um eine Verbindung zu Cloud Volumes ONTAP über ONTAP System Manager oder die ONTAP CLI herzustellen. Sie können den Standardbenutzernamen _admin_ beibehalten oder ihn durch einen benutzerdefinierten Benutzernamen ersetzen.
. *Tags*: Tags sind die Metadaten Ihrer AWS- oder Azure-Ressourcen. Die Konsole fügt die Tags der Cloud Volumes ONTAP Instanz und jeder zugehörigen Ressource hinzu. Sie können beim Erstellen eines Cloud Volumes ONTAP Systems bis zu 15 Tags über die Benutzeroberfläche hinzufügen und anschließend weitere hinzufügen. Weitere Informationen finden Sie unter:

.Bereitstellung und Konfiguration
. *Bereitstellungstyp/Topologie*: Wählen Sie den Bereitstellungstyp aus, den Sie verwenden möchten: Einzelknoten, Hochverfügbarkeit (HA) in einer einzelnen Verfügbarkeitszone (AZ) oder HA in mehreren AZs.
. *Netzwerkkonfiguration*: Geben Sie die Netzwerkinformationen ein, die Sie im Planungsarbeitsblatt für Regionen, VNets und Subnetze erfasst haben.
+
.. *Region*: Standardmäßig ist die Region des zugehörigen Cloud-Kontos mit den Ressourcen ausgewählt.
.. *Verfügbarkeitszone*: Sie können die Verfügbarkeitszone basierend auf dem von Ihnen gewählten Bereitstellungstyp auswählen, und wenn die ausgewählte Region die Verfügbarkeitszone unterstützt.
.. *Subnetz*: Wählen Sie ein Subnetz für die Region aus.
+
[role="tabbed-block"]
====
.Für AWS
--
*VPC*: Geben Sie für AWS eine VPC für die Region mit einem Subnetz gemäß Ihrer Planung ein.

.AWS-Hochverfügbarkeit
Wenn Sie eine HA-Konfiguration ausgewählt haben, geben Sie die folgenden Informationen ein:

.HA in einzelner AZ
... *Mediatorzugriff*: Geben Sie die Mediatorzugriffsinformationen an.  Der Mediator ist eine separate Instanz, die den Zustand des HA-Paares überwacht und im Fehlerfall ein Quorum bereitstellt.  Geben Sie den Namen des Schlüsselpaars an, damit die Mediatorinstanz eine Verbindung mit dem AWS EC2-Dienst herstellen kann, und wählen Sie die Verbindungsmethode aus.


.HA in mehreren AZ
... *Verfügbarkeitszonen und Mediator*: Wählen Sie die Verfügbarkeitszonen (AZs) für jeden Knoten und den Mediator sowie die entsprechenden Subnetze aus, in denen Sie das Cloud Volumes ONTAP HA-Paar bereitstellen möchten.
... *Floating IPs*: Wenn Sie mehrere AZs auswählen, geben Sie die Floating-IP-Adressen für die NFS- und CIFS-Dienste sowie die Cluster- und SVM-Verwaltung an.  Die IP-Adressen müssen für alle VPCs in der Region außerhalb des CIDR-Blocks liegen.  Weitere Einzelheiten finden Sie unterlink:https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/reference-networking-aws.html#requirements-for-ha-pairs-in-multiple-azs["AWS-Netzwerkanforderungen für Cloud Volumes ONTAP HA in mehreren AZs"^] .
... *Mediatorzugriff*: Geben Sie die Mediatorzugriffsinformationen an.  Der Mediator ist eine separate Instanz, die den Zustand des HA-Paares überwacht und im Fehlerfall ein Quorum bereitstellt.  Geben Sie den Namen des Schlüsselpaars an, damit die Mediatorinstanz eine Verbindung mit dem AWS EC2-Dienst herstellen kann, und wählen Sie die Verbindungsmethode aus.
... *Routentabellen*: Wenn Sie mehrere AZs ausgewählt haben, wählen Sie die Routentabellen aus, die Routen zu den Floating-IP-Adressen enthalten.  Wenn Sie mehr als eine Routentabelle haben, ist es wichtig, die richtigen Routentabellen auszuwählen.  Andernfalls haben einige Clients möglicherweise keinen Zugriff auf das Cloud Volumes ONTAP HA-Paar.  Weitere Informationen zu Routentabellen finden Sie im http://docs.aws.amazon.com/AmazonVPC/latest/UserGuide/VPC_Route_Tables.html["AWS-Dokumentation: Routentabellen"^] .

ifdef::azure[]

In Azure werden im Abschnitt *Übersicht* die Ressourcen angezeigt, die nach Abschluss des Bereitstellungsprozesses zugewiesen werden.

ifdef::aws[]

In _AWS_ startet die NetApp Console unmittelbar nach der Erstellung des Cloud Volumes ONTAP Systems eine Testinstanz in der angegebenen VPC, um die Konnektivität zu überprüfen. Bei Erfolg beendet die Console die Instanz sofort und beginnt dann mit der Bereitstellung des Systems. Wenn die Console die Konnektivität nicht überprüfen kann, schlägt die Erstellung des Systems fehl. Die Testinstanz ist entweder eine `t2.nano` (für Standard-VPC-Tenancy) oder eine `m3.medium` (für dedizierte VPC-Tenancy).

endif::aws[]

.Nach Abschluss
Die Konsole stellt das Cloud Volumes ONTAP -System bereit.  Sie können den Fortschritt auf der Audit-Seite verfolgen.

Sollten bei der Bereitstellung des Cloud Volumes ONTAP Systems Probleme auftreten, überprüfen Sie die Fehlermeldung. Sie können außerdem die Option *Umgebung neu erstellen* auf dem System auswählen, um die Bereitstellung mithilfe des  `Advanced`-Modus-Assistenten fortzusetzen.


CAUTION: Nach Abschluss des Bereitstellungsprozesses dürfen die systemgenerierten Cloud Volumes ONTAP Konfigurationen im AWS- oder Azure-Cloudportal, insbesondere die System-Tags, nicht verändert werden. Jegliche Änderungen an diesen Konfigurationen können zu unerwartetem Verhalten oder Datenverlust führen.
